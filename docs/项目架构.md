# CowAgent 项目架构

CowAgent 是一个基于大模型的超级 AI 助理，采用**"渠道为前端 + Agent 为后端"**的分层架构设计。

## 架构概览

```
┌─────────────────────────────────────────────────────────┐
│                    Channel (前端/渠道层)                  │
│  channel/                                              │
│  - web, feishu, dingtalk, wechat, wechatmp, wework,  │
│    terminal 等各种渠道实现                              │
│  职责：接收消息、发送消息、与各平台交互                   │
└─────────────────────────────────────────────────────────┘
                           ↓↑
┌─────────────────────────────────────────────────────────┐
│              Bridge (桥接层) + ChatChannel              │
│  bridge/                                              │
│  - Bridge: 管理不同类型的 bot 和模型                   │
│  - AgentBridge: 将 Agent 系统与 COW 桥接               │
│  - ChatChannel: 通用消息处理逻辑                        │
└─────────────────────────────────────────────────────────┘
                           ↓↑
┌─────────────────────────────────────────────────────────┐
│                  Agent (本体/后端核心)                   │
│  agent/                                               │
│  - protocol/agent.py: 核心 Agent 类                    │
│  - tools/: 各种工具 (browser, bash, edit, read,       │
│            write, scheduler, memory, web_search 等)    │
│  - skills/: 技能系统                                   │
│  - memory/: 长期记忆管理                                │
│  - prompt/: 提示词构建                                  │
└─────────────────────────────────────────────────────────┘
                           ↓↑
┌─────────────────────────────────────────────────────────┐
│                  Models (模型层)                        │
│  models/                                              │
│  - openai, claude, gemini, minimax, zhipu, dashscope,│

└─────────────────────────────────────────────────────────┘
```

## 1. Channel（渠道层）- 前端

### 职责

负责与各种外部平台交互，接收用户消息、发送回复。

### 支持的渠道类型

| 渠道类型 | 说明 | 配置文件参数 |
|---------|------|-------------|
| web | 网页控制台 | `channel_type: "web"` |
| terminal | 终端 | `channel_type: "terminal"` |
| feishu | 飞书 | `channel_type: "feishu"` |
| dingtalk | 钉钉 | `channel_type: "dingtalk"` |
| wechatmp | 个人公众号 | `channel_type: "wechatmp"` |
| wechatmp_service | 企业公众号 | `channel_type: "wechatmp_service"` |
| wechatcom_app | 企业微信应用 | `channel_type: "wechatcom_app"` |
| wework | 企业微信 | `channel_type: "wework"` |
| wx | 微信 (itchat) | `channel_type: "wx"` |
| wxy | 微信 (Wechaty) | `channel_type: "wxy"` |
| wcf | 微信 (WechatFerry) | `channel_type: "wcf"` |

### Web 渠道特殊功能

- 提供完整的 Web 控制台（默认端口 9899）
- SSE 流式响应支持
- REST API：消息发送、轮询、配置管理
- 静态资源服务
- 工具/Skills/记忆/定时任务管理等 UI

## 2. ChatChannel（消息处理层）

### 职责

- 消息过滤（关键字匹配、前缀匹配、黑名单）
- Context 构造（群聊/私聊识别、session 管理）
- 消息队列消费
- 插件事件触发
- 回复的装饰和发送

### 关键方法

```python
# 构造 Context
def _compose_context(self, ctype, content, **kwargs):
    """根据消息构造 Context"""

# 处理消息
def _handle(self, context: Context):
    """处理消息，构建回复"""

# 生成回复
def _generate_reply(self, context: Context, reply: Reply) -> Reply:
    """调用 Bridge 获取回复内容"""

# 装饰回复
def _decorate_reply(self, context: Context, reply: Reply) -> Reply:
    """装饰回复，如添加群聊 @、前缀后缀等"""

# 发送回复
def _send_reply(self, context: Context, reply: Reply):
    """发送回复到用户"""
```

## 3. Bridge（桥接层）

### 职责

- 管理多种 bot 类型（chat、voice_to_text、text_to_voice、translate）
- 根据配置选择具体模型实现
- 提供统一的对话接口
- 支持 Agent 模式切换

### 核心代码

```python
class Bridge(object):
    def __init__(self):
        self.btype = {
            "chat": const.CHATGPT,
            "voice_to_text": conf().get("voice_to_text", "openai"),
            "text_to_voice": conf().get("text_to_voice", "google"),
            "translate": conf().get("translate", "baidu"),
        }
    
    def fetch_reply_content(self, query, context: Context) -> Reply:
        """获取对话回复"""
        return self.get_bot("chat").reply(query, context)
    
    def fetch_agent_reply(self, query, context, on_event, clear_history) -> Reply:
        """获取 Agent 回复"""
        agent_bridge = self.get_agent_bridge()
        return agent_bridge.agent_reply(query, context, on_event, clear_history)
```

## 4. AgentBridge（Agent 集成层）

### 职责

- 将 Agent 系统与 COW 桥接
- 管理多会话 Agent 实例（每个 session_id 独立）
- 消息持久化
- 工具过滤（定时任务场景排除 scheduler）

### 核心功能

```python
class AgentBridge:
    def get_agent(self, session_id: str = None) -> Optional[Agent]:
        """获取 Agent 实例，每个 session_id 独立"""
    
    def agent_reply(self, query, context, on_event, clear_history) -> Reply:
        """使用 Agent 回复"""
    
    def refresh_all_skills(self) -> int:
        """刷新所有 Agent 实例的 Skills（热更新）"""
```

## 5. Agent（核心推理引擎）

### 核心组件

- **LLMModel**：模型适配器，调用各种大模型
- **ToolSystem**：工具系统
- **SkillManager**：技能管理器
- **Memory**：长期记忆管理

### Agent 核心特性

- **多轮任务决策**：能够理解复杂任务并自主规划执行
- **长期记忆**：自动将对话记忆持久化，支持关键词及向量检索
- **工具调用**：支持访问文件、终端、浏览器、定时任务等系统资源
- **Skills 系统**：内置多种技能，支持通过自然语言对话完成自定义 Skills 开发

## 8. 完整消息流程

```
用户发送消息
    ↓
Channel 接收（Web/飞书/钉钉/微信等）
    ↓
ChatChannel._compose_context() 构造 Context
    ↓
消息队列 → ChatChannel._handle()
    ↓
判断 agent 模式（config 中 agent=true/false）
    ↓
【Agent 模式】
AgentBridge.get_agent() 获取/创建 Agent
Agent.run_stream() 执行推理
  ↓ Agent 自主决策循环
  - LLMModel.call() 调用大模型
  - Agent 决定使用 Tools
  - ToolManager 执行工具
  - 结果返回 LLM 继续推理
  ↓
返回 Reply
    ↓
【普通模式】
Bridge.fetch_reply_content() 直接获取回复
    ↓
ChatChannel._decorate_reply() 装饰回复
    ↓
ChatChannel._send_reply() 发送回复
    ↓
Channel.send() 发送到对应平台
```

## 9. 配置说明

```json
{
    "channel_type": "web",
```

## 总结

CowAgent 项目采用典型的分层架构：

1. **Channel（渠道层）** = 前端：负责与各种外部平台交互
2. **Bridge + ChatChannel** = 桥接层：协调各组件工作
3. **Agent** = 后端核心：负责任务规划、工具调用、记忆管理
4. **Models** = 模型层：各种大模型的接口适配

这种架构设计具有良好的扩展性，可以方便地：
- 接入新的渠道（只需实现 Channel 接口）
- 添加新的模型支持（只需实现对应的 Bot 类）
- 扩展新的工具（只需继承 BaseTool）
- 开发新的 Skills
